1. Descripción general del sistema

Aplicación web desarrollada en Next.js 16 (App Router) para hacer seguimiento al proceso de virtualización de cursos académicos en la Corporación Universitaria Americana.

ComponenteTecnología
FrameworkNext.js 16.2.6 con App Router
FrontendReact 19 + Tailwind CSS 4
AutenticaciónNextAuth v4 (Credentials provider)
Base de datos principalGoogle Sheets (sincronizado con Excel local)
Base de datos local respaldoArchivo Excel data/cursos.xlsx
CorreosGoogle DWD → OAuth Personal → SMTP (en cascada)
DespliegueLinux + Nginx en /seguimiento + PM2
URL de producciónhttp://kora.americana.edu.co/seguimiento

7. Correcciones técnicas (v2.0)

7.1 Normalización de nombres de columna al leer Excel FIX

readSheet() ahora aplica normalizeRowKeys() después de sheet_to_json. Esto mapea variantes de nombres de columna (ej: "Link " con espacio, "Enlace", "Gestor responsable ") a sus nombres canónicos usando COL_ALIASES. Antes, curso.Link era undefined si la columna en el Excel tenía un espacio al final.

7.2 Claves de course-links.json con espacios FIX

Las claves del archivo data/course-links.json se normalizan al cargar y al guardar (trim de cada segmento separado por ::). Antes, una clave guardada como "Doctorado::Doctorado en Desarrollo Sostenible::Electiva Disciplinar III " (espacio al final) nunca coincidía con la búsqueda que usaba .trim().

7.3 Comparación de fechas en reporte FIX

El reporte de aprobados usaba String(fecha) para comparar, pero las fechas del Excel llegan como objetos Date (por cellDates: true). Se añadió la función parseDateField() que normaliza cualquier formato (Date, serial Excel, ISO, DD/MM/YYYY) a DD/MM/YYYY antes de comparar.

7.4 DWD con adjuntos (MIME multipart) FIX

sendViaDWD() ahora construye un mensaje multipart/mixed cuando hay adjuntos, en lugar de un email HTML simple de una sola parte. Esto permite enviar el Excel adjunto vía DWD sin necesitar SMTP.

7.5 Formato de observaciones en correos FIX

Las observaciones escritas en formularios (con saltos de línea, párrafos) ahora llegan correctamente al correo. Cada línea se envuelve en un <p> HTML, preservando el formato original.

7.6 Badges de días eliminados MEJORA UX

Los badges de días transcurridos (ej: 664786d) fueron eliminados de las vistas de Gestor, DI y Admin. La información de fechas y tiempo transcurrido está disponible en el modal de detalle del Admin.

9. Configuración de correos — Google DWD

El sistema envía correos usando Domain-Wide Delegation (DWD) de Google. Esto permite que los correos salgan con el nombre real de quien tomó la acción (ej: el correo aparece enviado por Samir Palencia aunque lo envíe un servidor).

Cadena de envío (en orden de prioridad)

#MétodoCondición
1Google DWDSi GOOGLE_SERVICE_ACCOUNT_EMAIL, GOOGLE_PRIVATE_KEY y fromEmail están configurados
2OAuth PersonalSi el usuario conectó su Gmail en /connect-gmail
3SMTPSi SMTP_USER y SMTP_PASS están configurados
Para el reporte automático: Se necesita REPORT_FROM_EMAIL en .env.local con una dirección del dominio @americana.edu.co que la cuenta de servicio pueda impersonar (DWD). Ejemplo: REPORT_FROM_EMAIL=coordinacion_gc@americana.edu.co

11. Instalación y ejecución local

Requisitos

  • Node.js v18+ instalado desde nodejs.org
  • Archivo .env.local configurado en la raíz del proyecto

Comandos

# 1. Instalar dependencias (solo la primera vez)
npm install --legacy-peer-deps

# 2. Iniciar servidor de desarrollo
npm run dev
# → Abre http://localhost:3000

# 3. Construir para producción
npm run build

# 4. Verificar errores de linting
npm run lint
Nota Windows: Si aparece error de permisos de PowerShell al correr npm, abre PowerShell como administrador y ejecuta:
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
O usa directamente: C:\Program Files\nodejs\npm.cmd run dev
